拓朴排序

拓扑排序的规则：

  1. 在有向图中选一个没有前驱的顶点并且输出
  2. 从图中删除该顶点和所有以它为尾的弧，即删除所有与它有关的边。
  3. 重复上述两步，直至全部顶点均已输出；或者当图中不存在无前驱的顶点为止。

BFS&DFS

回忆一下之前的印象笔记【[LeetCode]785. Is Graph Bipartite?】，当时就提到了BFS和DFS。有一点点难
这题就更进阶了，要求你：

  1. 建一个有向图（注意，是有向)
  2. 遍历这个图，按拓扑排序的规则依次输出
  3. 如果不能全部输出，那就有问题（这个有向图中存在环路）

其中遍历的这个部分，就可以用到BFS和DSF

range
函数说明：range([start,] stop[, step])，根据start与stop指定的范围以及step设定的步长，生成一个序列。

xrange
函数说明：用法与range完全相同，所不同的是生成的不是一个数组，而是一个生成器。

结论：对大数列来讲，xrange会消耗更少的内存

解题

思路

BFS

具体执行过程
用一个队列存入度为0的节点，依次出队，将与出队节点相连的节点的入度减1，如果入度减为0，将其放入队列中，直到队列为空。如里最后还有入度不为0的节点的话，说明有环，否则无环。

DFS

DFS思路

  1. if node v has not been visited, then mark it as 0.
  2. if node v is being visited, then mark it as -1. If we find a vertex marked as -1 in DFS, then their is a ring.if node v has been visited, then mark it as 1.
  3. If a vertex was marked as 1, then no ring contains v or its successors.

需要一个一维数组visit来记录访问状态，这里有三种状态，0表示还未访问过，1表示已经访问了，-1表示正在访问，如果在遍历过程中找到标记为-1的点，说明有环。

大体思路是，先建立好有向图，然后从第一个门课开始，找其可构成哪门课，暂时将当前课程标记为已访问，然后对新得到的课程调用DFS递归，直到出现新的课程已经访问过了，则返回false，没有冲突的话返回true，然后把标记为已访问的课程改为未访问。
