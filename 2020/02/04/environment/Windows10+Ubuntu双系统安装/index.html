Windows10+Ubuntu双系统安装

[toc]

攻略

Windows10+Ubuntu双系统安装 本身有问题，重点看评论
在win10下安装Linux双系统 这个是比较靠谱的
双系统安装Linux的步骤以及一些误区 有注意事项

准备工作

硬件：

  • 一个U盘(>2G)
  • 一台windows电脑

软件：

  • ubuntu系统，推荐16.04
  • UltraISO

数据备份

该上传上传，该存进硬盘存进硬盘

Windows准备空间

  1. 进入磁盘管理（快捷键Win + X)
  2. 选择剩余空间较大的可分配磁盘，右键并选择“压缩卷”，会出现黑色的未分配空间。
    未分配空间即用来装ubuntu的空间。越大越好，但通常40G就够用
  3. 有个比较奇怪但大家都要求的操作：在未分配空间上“新建卷”，再“删除卷”可以将这变成可用空间

制作Ubuntu启动盘

  1. 插入U盘。该U盘会被格式化，如果有重要内容需要提前备份
  2. 打开UltraSO软件，选择“文件”-“打开”打开之前下好的iso映像文件
  3. 选择“启动”-“写入硬盘映像”。此处注意：硬盘驱动器的选择是否正确；写入方式选择USB-HDD+
  4. 依次点击“格式化”和“写入”，完成后它会在消息处提示“刻录成功！”

安装ubuntu

  1. 插入U盘，将Windows电脑重启，进入BIOS系统（我的是F2），选择从U盘启动。
    我的从U盘启动有两种供选，一个是UEFI：USBXXXX，一个是USBXXXX，都是那个U盘。当我选择UEFI：USBXXXX进入系统后有提示会出现和windows不兼容的情况。所以建议是从USBXXXX这种传统方式进入安装模式
  2. 一步步执行，建议执行过程中不联网。语言建议选英语。下图是个错误示范：安装中下载更新和安装这个第三方软件建议不要勾选。（联网要几十分钟，不联网安装只需几份。完全可以安装完再更新）
    毕竟我们想要的是Linux开发环境，又不是想真的拿它当个系统用
  3. 安装类型，非常重要的一步，选择“其他选项”
  4. 至关重要的步骤， 给Ubuntu分配空间。点击未分配空间（一般会显示为空闲），点击左下角”+“，依次分配空间。
  • swap 逻辑分区，4G，分区格式swap（交换分区）
    大小为计算机的内存的1-2倍，8G就够，16G浪费
  • / 主分区，大小30G-40G，分区格式ext4
    安装系统以及软件的默认下载地点
  • /home 逻辑分区，大小为剩下的全部，分区格式ext4
    存放音乐、图片及下载等文件的空间，越大越好
  • 启动引导器，选择默认，不要像教程那样选择/boot所在的分区
  1. 安装等待。
  2. 出现重启即为安装成功。

说明
/boot：一般的攻略都是让建这个分区，把ubuntu引导安装在/boot下，再通过软件easyBCD设置引导，这样便可以从windows引导进入ubuntu。
这样导致两个问题：多了一个额外的easyBCD引导步骤，工作量加大；可能会出现引导失败的情况，这个涉及到基于UEFI下的引导，mbr和传统bios引导（我就失败了）。考虑到从ubuntu的引导进windows挺好的，所以这个分区建议不要建

注意事项

  1. 启动引导器一定选择默认的磁盘，或者必须是主引导盘，不要设成/boot
  2. 若想卸载Ubuntu，切记不能直接在Windows系统下直接格掉给Ubuntu的磁盘空间，否则两个系统都进不了；卸载方法为：下载MbrFix软件，对Mbr先进行修复，然后再格盘！(64位系统选择MbrFix64软件）
    Ubuntu/Win10双系统安全删除Ubuntu的方法

    可能出现的问题

    Linux安装程序出现：该分区不可用

windows系统硬盘分区有两种方式，一种是bios+MBR，一种是uefi+GPT，如果是后者那就可以直接分盘，如果是前者，当你有三个盘或以上时就会出现如下问题：分配后看上去一切都很成功，但是在进入Linux安装程序中会出现该分区不可用的情况。
这是因为在MBR格式中只允许四个主分区存在，压缩出来的盘不能创建为卷，也就不能被系统识别。
关于如何将MBR格式转为GPT格式，网上有许多的方法，基本上需要格式硬盘重装系统，或者借助一些工具。也可以选择简单粗暴的方式，直接删掉了一个盘。

电脑启动时为黑屏且左上角为光标闪烁

－ 未及时拔下U盘
－ boot顺序没有更改
－ 没有更改为UEFI　ONLY
－ boot对应的硬盘选择错误，电脑没找到可以启动的系统文件

电脑启动时为黑屏且显示grub

  • 安装了多个系统
  • 启动引导盘设置出错，电脑没有找到引导盘
